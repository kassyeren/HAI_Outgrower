## 🏗️ 1. Core Architecture & Tech Stack

### Frontend Architecture (Vanilla++ Lifecycle)

* **Base Framework:** Standard compliant semantic HTML5 and modern **Vanilla JavaScript**. No heavy client-side builds (React/Vue omitted to keep runtime footprints minimal).
* **Styling Engine:** **Tailwind CSS Engine v4** loaded dynamically via CDN, utilizing custom branding extensions and absolute positioning layers.
* **Mapping Runtime:** **MapLibre GL JS (v3.6.2)** integrated with standard open basemaps (`CartoDB.Positron`). Handles programmatic canvas recalculations on UI layout resizing.
* **Database Broker:** Direct client-side consumption via the **Supabase JavaScript Client SDK**.

### Backend & Database Layer (Supabase Architecture)

* **Relational Tables:**
* `farmer_profiles`: Primary index mapping master agricultural records.
* `farm_parcels`: Spatial asset table managing transactional data attributes and geometric coordinates.
* `farmer_documents`: Document tracking index.


* **Database View:** Data aggregation served natively through `vw_farm_dashboard` to streamline analytical tracking requests.
* **Storage Framework:** Dedicated binary asset bucket named `farmer_id_cards` storing secure authentication image profiles.
